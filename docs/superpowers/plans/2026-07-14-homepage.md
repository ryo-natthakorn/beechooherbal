# Homepage Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the real EN (`/`) and TH (`/th/home/`) homepages for beechooherbal.com, replacing
the Phase 2 placeholders, using the brand system from `2026 BC_brandguide_R4_18122025.pdf` and
content pulled verbatim from the live site's WP REST export.

**Architecture:** Static Astro pages (`src/pages/index.astro`, `src/pages/th/home.astro`) built
from shared components (`Header`, `Footer`, `TreatmentCard`) and a shared data file
(`src/data/treatments.ts`) that holds the 7 treatments' bilingual copy, image, and correct link.
Both pages route through the existing `BaseLayout` → `SEOHead` so hreflang keeps coming from
`src/i18n/pairs.ts` with no changes needed there. All images live in `src/assets/images/` and are
rendered with Astro's `<Image>` component (not raw `<img>`) so they get real optimization.

**Tech Stack:** Astro 5, Tailwind CSS v4 (`@theme` tokens, no config file), `@fontsource/roboto`
and `@fontsource/noto-sans-thai` for self-hosted fonts, Node/curl for one-time asset download.

## Global Constraints

- English lives at `/`, Thai at `/th/home/` — do not change either URL (CLAUDE.md §6, Path A).
- Thai copy must be used verbatim from the live site — no machine translation, no invented claims
  (CLAUDE.md §8).
- Every image needs real `alt` text and must go through Astro's `<Image>` component, never a raw
  `<img>` tag (CLAUDE.md §7).
- Known bug to fix, not copy: treatment card **headings** currently link to
  `beechooladies.com.sg`; both the heading and the "Read more" button must link to the correct
  `beechooherbal.com` (or `beechooherbal.com/th/...`) page (CLAUDE.md §1, §7).
- Downloaded images must be validated by magic bytes before use — Cloudflare can return a 200
  with HTML instead of the actual image (CLAUDE.md §7).
- No women-only messaging; this clinic serves everyone (CLAUDE.md §1, §8).
- Both pages must be wired through `BaseLayout` with `title`/`description`/`lang` so hreflang
  comes from `src/i18n/pairs.ts` (CLAUDE.md §6).

## Content decision made during planning (flag for Crispin)

The live homepage's Reviews section uses two ambiguous, unlabeled WP media assets
(`Bee-Choo-awards.jpg`, four repeats of `Smiling-Face-Emoji-300x300.png`) with no alt text and no
clear role. Rather than guess at their placement, this plan renders the Reviews section as the
real heading ("REVIEWS ON BEECHOO HAIR TREATMENT" / TH equivalent) plus a link to
`/reviews-and-testimonials-of-bee-choo-origin-treatment/`, which will carry the actual testimonial
content when that page is built later in Phase 3. This preserves the verified textual content
without fabricating a layout for images whose purpose isn't verifiable. Flag this simplification
to Crispin alongside the homepage sign-off.
