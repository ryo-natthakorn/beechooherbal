# Orphan Reconciliation — independent re-derivation

Generated by re-deriving everything from the **raw data only** (`pages.json`, `old-urls.txt`,
`en-th-map.csv`, `rest-pages.json`). `orphans-for-crispin.md` was treated as unverified and
its numbers were ignored until the final comparison. The analysis was run with a fresh,
self-contained script (no reuse of the inventory's own orphan logic), so this is a genuine
cross-check rather than a re-run of the same code.

## Method — the exact rules used

**Language of a page** (`langOf`): a URL is **Thai** if ANY of these hold, else **English**:
1. its path starts with `/th/`, OR
2. its `og:locale` starts with `th`, OR
3. its `<html lang>` starts with `th`, OR
4. the decoded path OR the `<title>` contains Thai-script characters (U+0E00–U+0E7F).

Rule 4 is the decisive one: Phase 1 established that `<html lang>` and `og:locale` both report
`en-US`/`en_US` on the Thai posts that sit at the root, so script-in-slug/title is what actually
reveals their language. `/th/` prefix is treated as definitional.

**Has a twin** (from `en-th-map.csv` only): a page "has a twin" if its normalized URL appears
in the CSV as `en_url` on a row whose `th_url` is non-blank, OR as `th_url` on a row whose
`en_url` is non-blank. Normalization = lowercase host + percent-decoded path + trailing slash +
drop query/hash (so the encoded URLs in `pages.json` match the decoded URLs in the CSV).
**EN-only** = English pages with no twin. **TH-only** = Thai pages with no twin.

## ITEM 2 — Male pattern baldness → **CONFIRMED CLEAN (genuinely English-only)**, one wording caveat

`/male-pattern-baldness-treatment-in-bangkok-causes-symptoms-of-hair-loss/`
(`title: "MALE PATTERN BALDNESS - Treatment in Bangkok, Causes, Symptoms of hair loss"`) is
**not in `en-th-map.csv`** and has **no Thai counterpart** in the data.

Searched every Thai page title for baldness-related terms
(`ล้าน`, `ศีรษะล้าน`, `หัวล้าน`, `ผมบาง`, `กรรมพันธุ์`, `พันธุกรรม`, `ฮอร์โมน`, `แอนโดรเจน`, `ผู้ชาย`):

  but that is the **main hair-loss treatment service page**, already paired with the English
  `/scalp-hair-loss-treatment-salon-clinic-in-bangkok/`. It is not a male-pattern-baldness article.
- The closest topical Thai *post* is `/โรคผมร่วงและทรีทเม้นท์/` ("โรคผมร่วงชนิดต่างๆ… ผมร่วงเป็นหย่อม…",
  various hair-loss diseases) — a general hair-loss article, not male-pattern-specific, and itself an
  unpaired Thai root post. The one explicitly-gendered Thai post is about **women**
  (`5 ตัวช่วยเรื่องผมร่วงของผู้หญิง`), i.e. the opposite audience.

So there is **no plausible dedicated Thai twin** — the broken hreflang did not "lose" a link here;
the Thai page simply does not exist. **Genuinely English-only.**

> Caveat on the old list's wording: it says this page "reads like a service page." Per
> `rest-pages.json`, it is **not** a WordPress *page* — it does not appear in the `pages` REST
> collection, so it is a **blog post** (post-sitemap). Topically it resembles a treatment article,
> but structurally it is a post. Minor mislabel; the English-only classification itself is correct.

**Verdict: CONFIRMED CLEAN** (English-only is correct; treat the "service page" description as
editorial, not structural).
